1.5:
Describing copyright and other constraints affecting websites

When building a website, particularly one bound for commercial use [3], it is necessary to bear in mind that many of the written works, images, themes, and even software [1] one may be tempted to include therein are subject to some form of copyright. It is an error of judgement to assume that software is not subject to the same protection as other works of intellectual property [2]. In the UK, at least, original content obtains copyright protection at the moment of creation [2], and there is no burden on the producer of said works to explicitly mark this [2]. Failure to take note of this can involve the developer and/or their business in legal difficulties and can incur penalties if considered serious enough [4].

When a developer is contemplating the use of content of which they themselves are not the author, they must refer to any available information provided by said author pertaining to the use of materials in order to avoid being in breach of the law.

1.6:
Describing access issues that need taking into account - describe issues that people with disabilities may face when using webpages and how we can overcome them

Public facing websites and applications can come to be used by a wide array of people; some of these people may, due to disability, face issues of accessibility; in order that these customers are able to make effective use of the software, the developer should investigate and implement software features to mitigate this [1]. To properly confront this issue, consideration should be given to the categories of disability and how each poses different barriers to use. Accessibility could also be said to entail considering the different types of device a person might be using. What size is the screen? Does it need to function with multiple languages? Will it make disproportionate demands on the batteries of mobile devices? Does the website deliver basic functionality without JavaScript and/or CSS? Should it? These are some of the questions it might be worth considering [2].

There are a number of techniques that can be utilised in providing accessible software to as wide a range of users as possible: choose colours that contrast in such a way that makes content easily readable for all people, including those with colour blindness; consider keyboard only users [3]; use HTML elements for content, not layout [3]. Screen readers are a variety of software used by the visually impaired to access on-screen content; they parse displayed content and attempt, as faithfully as they can, to re-render it in braille or via audio description [5]. There are resources on the web for testing how a given website interacts with different accessibility software; for example, silktide.com [4] provides a range of disability simulators with which developers can test their finished product before release to market.

Modern HTML5 features a number of new elements known as semantic elements; these do not generally alter the on-screen appearance or placement of content; however, they are designed in such a way that accessibility software can interact with them and, in this way, make more sense of the document in question and construct a logical picture of the document’s layout and progression for the user [6].

1.7:
Describe which are the appropriate filetypes for websites

A website can feature a wide array of different file types; the content of web pages is generally described by HTML (Hyper Text Markup Language) [1]; this is then parsed by the web browser to populate the page according to the contents listed therein [1]; these files use the extension “.html”. The style of a document is the responsibility of CSS (Cascading Style Sheets) [2]; this can be embedded within the HTML document itself (“inline” [3]); however, best practice prefers that this not be done [3]; CSS files are parsed by the browser and help it decide how to display the associated HTML document’s content.

JavaScript is a scripting language [8] which has the distinction amongst programming languages of being the most widely supported by major browsers; it is used to implement dynamic behaviour and functionality within HTML documents which would otherwise consist of static content [4]; JavaScript can be included within a HTML document by its insertion within “<script>” tags or by referencing the URL at which the script is located [5].

In addition to the above three file types, a website may feature a whole array of others relating to its multimedia content - eg: “.mp4”, “.mp3”, “.png”, “.jpg”, etc [6]. Whilst there has been a push for increasing standardisation, it is still difficult to ensure that a given technology can be used on all platforms; one way of eluding this particular risk is to make continuous use throughout the development cycle of online sources indicating whether the file type in question can rendered by the target browser; caniuse.com is an example of this type of resource [7].
